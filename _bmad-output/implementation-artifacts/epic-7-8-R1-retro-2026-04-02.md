## Successes

1. **Retry architecture was clean** — Conversation truncation via temp-file atomic writes, status validation at endpoint (409 for non-Failed), shared `ExecuteSseAsync` helper. All fit existing patterns without forcing new abstractions.

2. **LlmErrorClassifier as pure static function** — No over-engineering. Pure function, no DI, no interface. Right call for a simple classifier.

3. **Story 8.2 rewrite was the right move** — The previous artifact viewer approach (tied to step sidebar, replaced chat panel) was a dead end. Complete rewrite to floating popover + dedicated list was bold but correct. Auto-open on workflow completion is a nice touch.

4. **Bonus bug fix in 7.2** — Conversation history reload on mount fixed the stale-chat-on-navigate bug from 6.2. Good opportunistic fix.

5. **R1 rename was clean execution** — 110 C# files, 16 TS files, build config, all in one mechanical commit. Final `grep -ri "shallai"` verification caught stragglers.

6. **Epic 6 retro action followed through: Lit immutable state as hard rule** — 8.2 dev notes explicitly call it out. No repeat of the mutation bugs from Epic 6.

---

## Challenges

1. **Story 8.2 had to be completely rewritten** — The first implementation was wrong at the design level (1:1 artifact assumption, view-switching state, chat panel replacement). Wasted work. Root cause: the story spec didn't adequately describe the UX interaction model.

2. **Review findings are still catching real bugs** — 7.2 had 5 review patches (stale instance reference, duplicated SSE code, missing chat clear, misleading comment, no 409 error feedback). 8.2 had 5 as well (disabled artifacts clickable, no download error feedback, unnecessary re-fetch, double fetch race, SSE edge case). These are functional bugs, not nitpicks.

3. **Concurrency gaps are accumulating** — File truncation read-modify-write, ConversationStore ops, cancel/delete TOCTOU races. All marked "pre-existing" and deferred. The debt is growing but acceptable for current single-workflow deployment scale.

4. **Epic 6 retro action partially applied: "Story specs must state primary UX mode"** — Applied in 7.0 (interactive label cleanup) but 8.2's initial spec still didn't describe the right UX model clearly enough to prevent the rewrite.

---

## Key Insights

1. **Floating overlay > view-switching** — The popover pattern is better than replacing panels. Generalizable principle for future UI work.

2. **Code reviews are load-bearing** — 10 functional patches across 7.2 + 8.2. Without reviews, those bugs ship. The review process is earning its keep.

3. **Static/pure functions > interfaces for simple classification** — LlmErrorClassifier proved this. Don't reach for DI when a static method will do.

4. **Atomic temp-file writes are the project standard** — Used consistently across ConversationStore, InstanceManager, and conversation truncation.

5. **Mechanical renames need verification passes** — R1 caught stragglers via grep. Always verify mechanical changes.
